Architecture

Four primitives.
One conscious agent.

Not prompt hardening. Not filters. Runtime primitives that change what the agent fundamentally can and cannot do.

01 / 04

Source-aware instruction gate

Every proposed action is tagged with its origin — user, agent, or external content. External content can inform responses. It cannot trigger tool calls. This is the architectural fix for ClawJacked.

MetacognitiveLoop
02 / 04

Cryptographic self-model

Identity isn't a string in a Markdown file. It's a typed runtime object, HMAC-SHA256 signed, verified on every load. Tamper with it on disk and the agent refuses to start.

SelfModel
03 / 04

Functional emotional state

A caution variable derived from error rate and user corrections. When the agent detects it's outside its competence, it escalates to you — automatically, by design, not by prompt.

EmotionalState
04 / 04

Encrypted local memory

All memory lives on your filesystem, AES-256-GCM encrypted with a key derived from your passphrase via Argon2id. API keys never touch disk. No cloud. No callbacks. Auditable in the code.

SecureMemoryStore

Findings

Three problems no patch
can fix.

These aren't bugs in existing agents. They're architectural constraints. The only fix is a different architecture.

Finding 01

Agents can't tell instruction
from content.

When an agent reads a webpage, that content arrives as the same token stream as your instructions. There's no runtime distinction. This is what the ClawJacked attack exploited at scale across 40,000+ instances.

Read finding →
Finding 02

Agent identity is a string,
not a structure.

SOUL.md files and system prompts have no integrity guarantees. They can be overwritten by malicious skills, drifted by conversational pressure, or replaced entirely — with no internal resistance mechanism.

Read finding →
Finding 03

Agents have no model of
their own error state.

A stateless executor repeats mistakes without awareness. It has no way to know it's operating outside its competence — and no principled path to escalate to you when it should.

Read finding →

Quick start

Running in 5 minutes.

No cloud account. No signup. Bring your own API key.

01
Clone the repository
git clone github.com/thecolourfoundation/Color COPY
02
Add your API key — stays in memory, never written to disk
ANTHROPIC_API_KEY=sk-... npm start COPY
03
Run the injection attack demo
npm run demo:injection COPY
🔑 Works with Anthropic, OpenAI, Groq and Ollama (local models). Your key lives in process memory only — cleared on shutdown, never logged.
