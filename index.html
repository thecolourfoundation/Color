Research Findings

Three problems
with how agents
exist today.

Existing open-source agent runtimes share structural vulnerabilities that no amount of patching can fix. They require architectural solutions.

01
Finding 01
Agents cannot distinguish instruction from content.
When an agent processes a document, email, or web page, those inputs arrive as the same token stream as user instructions. The agent has no architectural mechanism to distinguish "my user asked me to do this" from "this document is asking me to do this."

This is not a model alignment problem. It is a runtime design problem. The fix requires source-tagged instruction tracking at the execution layer — not prompt engineering.
Read full finding →
02
Finding 02
Agent identity is a string, not a structure.
Current agent runtimes store identity as plaintext — a SOUL.md file, a system prompt string, a Markdown document. These have no integrity guarantees. They can be overwritten by malicious skills, modified by indirect injection, or drifted by conversational pressure.

A sovereign agent requires a cryptographically signed self-model: a runtime object the agent can verify, consult, and protect.
Read full finding →
03
Finding 03
Agents have no model of their own error state.
A stateless executor makes the same mistake repeatedly without internal awareness. It has no mechanism to detect that it is operating outside its competence, and no principled path for escalating to human oversight when it should.

We propose functional emotional state as a lightweight solution: a caution variable derived from error rate and correction frequency that dynamically raises the threshold for autonomous action.

Implementation

Colors.
The research,
running.

Colors is our reference implementation of the consciousness primitives described in TCF-2026-001. It is not a product. It is a working proof of concept — open source, auditable, and designed to be studied as much as used.

Every architectural claim in our research is verifiable in the codebase. The MetacognitiveLoop, SelfModel, EmotionalState, and SecureMemoryStore are each independently testable. 49 tests pass from a clean clone.

It runs entirely on your hardware. No cloud dependency. No telemetry. No remote memory. Your API key lives in process memory and is cleared on shutdown.

Independent research
into sovereign AI.

The Colour Foundation is an independent research group. We study the conditions under which AI agents can be trusted to operate autonomously — and the specific technical primitives required to make that trust warranted rather than assumed.

Our work is open by default. Every finding we publish is accompanied by a working implementation. We believe research that cannot be reproduced is not research.

We welcome collaborators, critical review, and independent replication of our results. If you have found a flaw in our security model or a gap in our reasoning, we want to know.
