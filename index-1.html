Why it matters

Three problems no patch
can fix.

These aren't bugs in existing agents. They're architectural. The only fix is a different architecture.

01

Your agent reads a page.
Gets hijacked by it.

Instructions hidden in webpages, documents, or emails look identical to your instructions inside the agent. It can't tell the difference — and attackers know this. 40,000+ agents were compromised this way in 2026.

02

Agent identity is a text file.
Anyone can rewrite it.

Most agents store their rules as plain text files. No signature. No verification. A malicious skill or document can overwrite who the agent thinks it is — and it won't know it happened.

03

When the agent fails,
you don't find out.

Stateless agents repeat the same mistakes silently. No awareness of their own error state. No mechanism to ask you for help when they should. No escalation path.

Architecture

Four primitives.
One conscious agent.

Not prompts. Not filters. Runtime structures that change what the agent can and cannot do — by design.

01 / 04

The instruction gate

Every action is tagged with where it came from — you, the agent, or external content. Content from the web cannot trigger tool calls. Ever. This closes the hijack attack class structurally.

MetacognitiveLoop
02 / 04

The signed self-model

Identity is a cryptographically signed runtime object — not a text file. If anyone tampers with it on disk, the agent refuses to start. Your agent stays who it's supposed to be.

SelfModel
03 / 04

The caution layer

When the agent detects it's making too many mistakes or is in uncertain territory, it raises a flag and asks before proceeding. Human oversight by design, not by accident.

EmotionalState
04 / 04

Encrypted local memory

Everything the agent remembers is AES-256-GCM encrypted on your machine. Your API keys live in process memory only — cleared on shutdown. Nothing leaves your device.

SecureMemoryStore

Findings

What we found.
What we published.

Three documented findings from TCF-2026-001. Read them, challenge them, build on them.

Finding 01

Agents can't tell instruction
from content.

When an agent reads a document, that content arrives as the same token stream as your instructions. No runtime distinction exists. This is the root of the ClawJacked attack class.

Read finding →
Finding 02

Agent identity is a string,
not a structure.

SOUL.md files and system prompts carry no integrity guarantees. They can be overwritten, injected, or drifted away by conversational pressure — with no internal resistance mechanism.

Read finding →
Finding 03

Agents have no model of
their own error state.

A stateless executor repeats mistakes without awareness. It has no way to know it's operating outside its competence — and no principled path to escalate to human oversight.

Read finding →
